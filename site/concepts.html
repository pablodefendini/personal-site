Three Visual Concepts

Each concept uses the same HTML. Just swap the CSS file in the <link> tag.

Concept A: Editorial Minimalist

css/style.css (default)

Clean, high-contrast, type-driven. Serif headlines (Georgia) on a warm off-white background. Generous whitespace, restrained palette — black text, subtle red hover accents, quiet rules between sections. The vibe is an independent literary magazine or a long-form journalism site.

This is the most classic option. It signals "writer and thinker who also designs" rather than "designer first." It lets the writing breathe.

To preview: open index.html — it uses this concept by default.

Concept B: Warm Brutalist

css/style-b.css

Raw, high-energy, utilitarian. Monospace body text (Courier) with bold Helvetica headlines in uppercase. A warm paper-toned background with a burnt orange accent color. Dashed borders, left-border markers on case study headers, visible structure. Inspired by zine culture, open-source aesthetics, and the kind of thing you'd build in a terminal.

This is the most opinionated option. It signals "I've shipped real things and I don't need polish to prove it." It has personality. Some people will love it, some won't — which is kind of the point.

To preview: change the CSS link in any HTML file from css/style.css to css/style-b.css

Concept C: Dark Mode Modernist

css/style-c.css

Dark background, system sans-serif type, soft blue link accents against warm gray text. Monospace timestamps. Minimal ornamentation — the dark background does the design work by making the type feel quiet and intentional. Inspired by developer tools, editorial dark modes, and reading apps.

This is the most "tech" option. It signals comfort with code and tools without looking like a developer portfolio. Good if the audience is primarily people in the civic tech and political technology space.

To preview: change the CSS link in any HTML file from css/style.css to css/style-c.css
